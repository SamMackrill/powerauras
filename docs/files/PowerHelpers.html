PowaAuras:CreateAura (page)
Creates a new aura with the default settings and type (Buff) on the given page. If a page is not given, then the currently selected page in the AuraEditor is used.

Parameters

  • page: The page to create the aura on. Leaving this blank will default to the selected page in the aura browser.
PowaAuras:CreateAuraFromImport (importString)
Creates a single aura or an entire aura set from the given import string. The type is determined automatically and if a set is to be imported then all auras on the currently selected page in the editor are deleted. If no page is selected, then it will default to the first page. If you are only importing a single aura, then it will attempt to find space on the currently selected page in the editor. If no space is found then false is returned, and importing is cancelled. It is recommended that you wrap any calls to this function inside of pcall() to prevent lua errors from being displayed to the user.

Parameters

  • importString: The string to read as an import code. The string is trimmed automatically.

Return value:

True if the import has probably succeeded, false otherwise.
PowaAuras:DeleteAura (id, quickDelete)
Attempts to delete the aura with the given ID. This will in turn invoke an aura reindexing after the aura has been deleted.

Parameters

  • id: The ID of the aura to delete.
  • quickDelete: Defaults to false, if set to true then ReindexAuras will not be called.

Parameters

PowaAuras:GetNextFreeSlot (page)
Determines the next free aura slot on the given page. If no page is specified then the selected page in the aura browser is used.

Parameters

  • page: The page to find a free slot on. If no page is given, the currently selected page is used.

Return values:

  1. A mixed value based on whether or not a free slot could be found on the page. An integer representing the empty aura ID slot if found, or false if no free slot was found on the page.
  2. The page index scanned. You can use this to determine the page scanned if you didn't pass one as a parameter.
PowaAuras:GetSetting (key, id)
Retrieves a setting with the associated key.

Parameters

  • key: The setting to be retrieved.
  • id: The ID of the aura to retrieve settings for, if the key represents an aura setting.

Return value:

Returns the value associated with the specified setting, or false if the setting does not exist.
PowaAuras:RegisterSetting (key, name, location)
Registers a saved variable as a setting, assigning it a key and allowing callbacks to be assigned to the setting. The key should be different from the actual variable name to prevent conflicts.

Parameters

  • key: The key to assign to the specified setting.
  • name: The name of the subkey in the location which represents this setting.
  • location: The location of the setting, use a value from PowaAuras.SettingLocations for this parameter.
PowaAuras:RegisterSettingCallback (func)
Registers a callback to be called whenever a setting is updated. Callbacks are supplied two parameters when called, which represent the key of the setting that was updated and the value that was set.

Parameters

  • func: The function to set as a callback. Callbacks cannot be unset.

Return value:

Returns true if the callback was successfully set, false if the callback already existed.
PowaAuras:ReindexAura (oldID, newID, doCopy)
Reindexes a specific aura from one ID to another, and updates the aura chains to reflect the change. By default the operation performed is a move, and the aura at the old location will be replaced with a nil value. Note that reindexing auras is a slow operation.

Parameters

  • oldID: The current location of the aura.
  • newID: The new location of the aura.
  • doCopy: Optional boolean, set to true if you want the aura to remain in the old location and to copy it to the new location. False is the default behaviour and moves the aura.
PowaAuras:ReindexAuras (fullCheck, checkPage)
Reindexes auras both internally and on the AuraBrowser interface, with an optional check to make sure no "gaps" are present.

Parameters

  • fullCheck: Set to true if you want a longer, more costly check to take place. This will move auras around to prevent gaps appearing. The full check is applied to all pages unless specified.
  • checkPage: Optional integer. Pass a specific page number to limit the reindexing operation to that specific page for a significant speed improvement.
PowaAuras:SaveSetting (key, value, id)
Saves the given value into a specific setting, and fires any callbacks associated with the setting in the process.

Parameters

  • key: The key of the setting to modify.
  • value: The value to be stored.
  • id: The ID of the aura to set settings for, if the key represents an aura setting.

Return value:

Returns true if the saving is successful, false otherwise.
PowaAuras:SettingExists (key)
Checks if the setting with the given key exists.

Parameters

  • key: The settings key to check for.

Return value:

Returns true if the setting exists, false otherwise.
PowaAuras:ToggleAllAuras (activeOnly, forceDisplay, state)
Toggles the displays of all auras. Can optionally force a specific display state, force a refresh or only update auras that are already actively displaying.

Parameters

  • activeOnly: Set to true if you only want to toggle auras that are actively displaying. Defaults to false.
  • forceDisplay: Set to true if you want to force the aura to refresh by toggling it off and then on immediately afterwards. Setting this to true will ignore the state parameter. Defaults to false.
  • state: Forces a specific state to be applied to the auras, set to true if you want the auras to show or false if you want the auras to hide. Nil values will toggle the state to be the inverse of what it is currently, and is the default.
PowaAuras:ToggleAuraDisplay (id, state, noUpdate)
Toggles the display state of an aura, hiding it if already shown and vice versa. Can optionally force a specific state to be used.

Parameters

  • id: The ID of the aura to toggle.
  • state: A state to force upon the aura. Pass true to force it to show, or false to force it to hide.
  • noUpdate: Set to true if you want the aura to display without triggering an update on the AuraBrowser.
PowaAuras:ToggleAuraEnabled (id, state)
Enables or Disables an aura. Can optionally use this to force a specific state on the aura.

Parameters

  • id: The ID of the aura to enable or disable.
  • state: An optional state to force on the aura. Set to true if you want the aura to be enabled, false otherwise.
PowaAuras:UpdateAuraTables (auraID)
Updates the global saved variable tables for Global/Class auras to reflect any changes to a specific aura ID or all aura ID's.

Parameters

  • auraID: The ID of the aura to use when updating a saved variable table. If omitted, then both the Global and Class tables are updated for all auras.
PowaAuras:UpdateSetting (key, value)
Fires all of the setting callbacks, passing the key of the setting that was updated and the new value to the functions.

Parameters

  • key: The key of the setting which has been updated.
  • value: The value to be passed to the callback functions.

Return value:

Returns false if the setting does not exist.
